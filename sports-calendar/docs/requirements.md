# sports-calendar requirements

## Functional
- Store sports events in a relational SQLite database.
- Import events from `sample_data/events.json` safely, even with nullable nested data.
- Keep enough structure for later list/get/create event APIs.

## Non-functional
- Simple, readable Python code suitable for a coding exercise.
- SQLAlchemy ORM models with explicit relationships.
- Minimal dependencies and easy local setup.
